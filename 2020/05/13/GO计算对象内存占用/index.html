2020-05-13

对于固定大小的对象可以使用encoding/binary包进行计算。

A fixed-size value is either a fixed-size arithmetic type (bool, int8, uint8, int16, float32, complex64, …) or an array or struct containing only fixed-size values.

在使用binary.Size(int)这样的值返回值是-1。

字节序

字节序 就是多字节数据类型在内存中的存储顺序。可分为大端序，低地址端存放高位字节；小端序与之相反，低地址端存放低位字节。

数字305424265=0x12346789

小端模式
[0]: 137 : 0x89 ——– 低地址
[1]: 103 : 0x67
[2]: 52 : 0x34
[3]: 18 : 0x12
[5]: 0 : 0x00
[6]: 0 : 0x00
[7]: 0 : 0x00
[8]: 0 : 0x00 ——– 高地址

大端模式
[0]: 0 : 0x00 ——– 低地址
[1]: 0 : 0x00
[2]: 0 : 0x00
[3]: 0 : 0x00
[5]: 18 : 0x12
[6]: 52 : 0x34
[7]: 103 : 0x67
[8]: 137 : 0x89 ——– 高地址

小端模式，编译时候，如果需要从int32->int64只需要增加内存范围，不需要做移动操作。
大端模式，主要是用于网络数据包传输，可以直接按顺序读取数据。

encoding/binary可变长编码

binary包中PutVarint使用的是可变长编码的方式来处理数据。

通常一个字节8位存储数据最大就是8个1=0xFF=255，变长编码使用第一位标识是否需要读取下个字节的内容。

比如

88 = 0x58 小于 127 直接存储在一个字节即可，因为不需要读取后面的字节内容。存储个常规的数字一样的。

234 = 0xEA

1110 1010 大于了127就需要2个字节来存储对应的数据。

10000001 –> 表示需要读取下个字节。

01101010 –> 上个最后一个1和7位合起来。
